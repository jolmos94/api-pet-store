📊 Load Test Report - Locust

🔹 Summary

A load test was conducted using Locust on the Petstore - OpenAPI 3.0 API to evaluate its performance under high concurrency conditions. During the test, it was observed that after approximately 2 minutes, with more than 50 simultaneous users, the API started returning 500 Internal Server Error, indicating server issues under load.

🚀 Test Details

  • Tool Used: Locust
  • Total Simulated Users: 50+ and 100
  • User Spawn Rate: 10 users per second
  • Test Duration: ~2 minutes until errors occurred
  • Evaluated Endpoints:
    • User
    • Store
    • Pet

🔍 Analysis of Results

🔸 Possible Causes of the 500 Error

  • Server Overload: Increased users may have exceeded processing capacity.
  • Simultaneous Connection Limits: Infrastructure restrictions may have blocked new requests.
  • Database Failures: Possible bottleneck in read/write operations.

✅ Recommendations

  • ✔️ Monitor the Server: Review logs and CPU, RAM, and network traffic metrics to identify bottlenecks.
  • ✔️ Optimize the API: Improve concurrency handling and database query efficiency.
  • ✔️ Detailed Errors: Configure more informative logs in the backend to diagnose the exact cause of the 500 errors.

📌 Conclusion

The test results revealed that the Petstore - OpenAPI 3.0 API does not effectively handle loads greater than 50 concurrent users. It is recommended to analyze and optimize the API to ensure better performance under high concurrency.
